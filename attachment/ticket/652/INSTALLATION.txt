_Building a 64-bit scientific python environment for FiPy from source: http://matforge.org/fipy/wiki/InstallFiPy/MacOSX/SnowLeopard</td></tr><tr><th id="L395"><a href="#L395">395</a></th><td>.. _Installing FiPy with pip: http://matforge.org/fipy/wiki/InstallFiPy/PipInstallsPython</td></tr><tr><th id="L396"><a href="#L396">396</a></th><td>.. _wiki: http://matforge.org/fipy</td></tr><tr><th href="#L399">399</a></th><td>.. _ENVIRONMENT:</td></tr><tr><th id="L400"><a href="#L400">400</a></th><td></td></tr><tr><th id="L401"><a href="#L401">401</a></th><td>-----------------------</td></tr><tr><th id="L402"><a href="#L402">402</a></th><td>Development Environment</td></tr><tr><th id="L403"><a href="#L403">403</a></th><td>-----------------------</td></tr><tr><th id="L404"><a href="#L404">404</a></th><td></td></tr><tr><th id="L405"><a href="#L405">405</a></th><td>It is often preferable to not formally install packages in the system</td></tr><tr><th id="L406"><a href="#L406">406</a></th><td>directories. The reasons for this include:</td></tr><tr><th id="L407"><a href="#L407">407</a></th><td></td></tr><tr><th id="L408"><a href="#L408">408</a></th><td> * developing or altering the package source code,</td></tr><tr><th id="L409"><a href="#L409">409</a></th><td></td></tr><tr><th id="L410"><a href="#L410">410</a></th><td> * trying out a new package along with its dependencies without</td></tr><tr><th id="L411"><a href="#L411">411</a></th><td>   violating a working system,</td></tr><tr><th id="L412"><a href="#L412">412</a></th><td></td></tr><tr><th id="L413"><a href="#L413">413</a></th><td> * dealing with conflicting packages and dependencies,</td></tr><tr><th id="L414"><a href="#L414">414</a></th><td></td></tr><tr><th id="L415"><a href="#L415">415</a></th><td> * or not having admin privileges.</td></tr><tr><th id="L416"><a href="#L416">416</a></th><td></td></tr><tr><th id="L417"><a href="#L417">417</a></th><td>The simplest way to use a :term:`Python` package without installing it</td></tr><tr><th id="L418"><a href="#L418">418</a></th><td>is to work in the base directory of the unpacked package and set </td></tr><tr><th id="L419"><a href="#L419">419</a></th><td>the :envvar:`PYTHONPATH` environment variable to "``.``". In order to work in an</td></tr><tr><th id="L420"><a href="#L420">420</a></th><td>directory other than the package's base directory, the :envvar:`PYTHONPATH`</td></tr><tr><th id="L421"><a href="#L421">421</a></th><td>environment variable must be set to ":file:`~/path/to/package`". This</td></tr><tr><th id="L422"><a href="#L422">422</a></th><td>method of working is adequate for one package, but quickly becomes</td></tr><tr><th id="L423"><a href="#L423">423</a></th><td>unmanageable with multiple :term:`Python` packages. In order to manage</td></tr><tr><th id="L424"><a href="#L424">424</a></th><td>multiple packages, it is better to choose a standard location other</td></tr><tr><th id="L425"><a href="#L425">425</a></th><td>than the default installation path. </td></tr><tr><th id="L426"><a href="#L426">426</a></th><td></td></tr><tr><th id="L427"><a href="#L427">427</a></th><td>If you do not have administrative privileges on your computer, or if</td></tr><tr><th id="L428"><a href="#L428">428</a></th><td>for any reason you don't want to tamper with your existing</td></tr><tr><th id="L429"><a href="#L429">429</a></th><td>:term:`Python` installation, most packages (including :term:`FiPy`)</td></tr><tr><th id="L430"><a href="#L430">430</a></th><td>will allow you to install to an alternative location.  Instead of</td></tr><tr><th id="L431"><a href="#L431">431</a></th><td>installing these packages with ``python setup.py install``, you would</td></tr><tr><th id="L432"><a href="#L432">432</a></th><td>use :samp:`python setup.py install --home={dir}`, where :samp:`{dir}`</td></tr><tr><th id="L433"><a href="#L433">433</a></th><td>is the desired installation directory (usually "``~``" to indicate</td></tr><tr><th id="L434"><a href="#L434">434</a></th><td>your home directory).  You will then need to append</td></tr><tr><th id="L435"><a href="#L435">435</a></th><td>:file:`{dir}/lib/python` to your :envvar:`PYTHONPATH` environment</td></tr><tr><th id="L436"><a href="#L436">436</a></th><td>variable.  See the `Alternate Installation`_ section of the</td></tr><tr><th id="L437"><a href="#L437">437</a></th><td>:term:`Python` document "`Installing Python Modules`_"</td></tr><tr><th id="L438"><a href="#L438">438</a></th><td>[InstallingPythonModules]_ for more information, such as circumstances</td></tr><tr><th id="L439"><a href="#L439">439</a></th><td>in which you should use :option:`--prefix` instead of</td></tr><tr><th id="L440"><a href="#L440">440</a></th><td>:option:`--home`.</td></tr><tr><th id="L441"><a href="#L441">441</a></th><td></td></tr><tr><th id="L442"><a href="#L442">442</a></th><td>.. _Alternate Installation: http://docs.python.org/inst/alt-install-windows.html</td></tr><tr><th id="L443"><a href="#L443">443</a></th><td></td></tr><tr><th id="L444"><a href="#L444">444</a></th><td>.. _Installing Python Modules: http://docs.python.org/inst/</td></tr><tr><th id="L445"><a href="#L445">445</a></th><td></td></tr><tr><th id="L446"><a href="#L446">446</a></th><td>An alternative to setting the :envvar:`PYTHONPATH` is to employ one of the</td></tr><tr><th id="L447"><a href="#L447">447</a></th><td>utilities that manage packages and their dependencies independently of</td></tr><tr><th id="L448"><a href="#L448">448</a></th><td>the system package manager and the system directories. These utilities</td></tr><tr><th id="L449"><a href="#L449">449</a></th><td>include Stow_, Virtualenv_ and zc.buildout_, amongst others. Here we'll</td></tr><tr><th id="L450"><a href="#L450">450</a></th><td>describe the use of Virtualenv_, which we highly recommend.</td></tr><tr><th id="L451"><a href="#L451">451</a></th><td></td></tr><tr><th id="L452"><a href="#L452">452</a></th><td>.. _Stow: http://savannah.gnu.org/projects/stow/</td></tr><tr><th id="L453"><a href="#L453">453</a></th><td>.. _zc.buildout: http://pypi.python.org/pypi/zc.buildout</td></tr><tr><th id="L454"><a href="#L454">454</a></th><td></td></tr><tr><th id="L455"><a href="#L455">455</a></th><td>Virtualenv</td></tr><tr><th id="L456"><a href="#L456">456</a></th><td>==========</td></tr><tr><th id="L457"><a href="#L457">457</a></th><td></td></tr><tr><th id="L458"><a href="#L458">458</a></th><td>http://pypi.python.org/pypi/virtualenv</td></tr><tr><th id="L459"><a href="#L459">459</a></th><td></td></tr><tr><th id="L460"><a href="#L460">460</a></th><td>Virtualenv_ enables the installation of packages in multiple isolated</td></tr><tr><th id="L461"><a href="#L461">461</a></th><td>environments. It organizes the installation of :term:`Python` packages</td></tr><tr><th id="L462"><a href="#L462">462</a></th><td>especially well and also provides a handy location for installing</td></tr><tr><th id="L463"><a href="#L463">463</a></th><td>non-:term:`Python` packages. In addition Virtualenv_ works seamlessly</td></tr><tr><th id="L464"><a href="#L464">464</a></th><td>with the :term:`PyPI` package manager (:term:`pip`).</td></tr><tr><th id="L465"><a href="#L465">465</a></th><td></td></tr><tr><th id="L466"><a href="#L466">466</a></th><td>The utility of Virtualenv_ is significantly augmented with Virtualenvwrapper_. </td></tr><tr><th id="L467"><a href="#L467">467</a></th><td></td></tr><tr><th id="L468"><a href="#L468">468</a></th><td>In general, the initial installation of Virtualenv_ and Virtualenvwrapper_</td></tr><tr><th id="L469"><a href="#L469">469</a></th><td>requires admin privileges, but thereafter, creating new virtual</td></tr><tr><th id="L470"><a href="#L470">470</a></th><td>environments and installing packages into them does not require admin</td></tr><tr><th id="L471"><a href="#L471">471</a></th><td>privileges.</td></tr><tr><th id="L472"><a href="#L472">472</a></th><td></td></tr><tr><th id="L473"><a href="#L473">473</a></th><td>.. _Virtualenvwrapper: http://www.doughellmann.com/projects/virtualenvwrapper</td></tr><tr><th id="L474"><a href="#L474">474</a></th><td></td></tr><tr><th id="L475"><a href="#L475">475</a></th><td>.. _documentation:GIT:</td></tr><tr><th id="L476"><a href="#L476">476</a></th><td></td></tr><tr><th id="L477"><a href="#L477">477</a></th><td>.. include:: documentation/GIT.txt</td></tr></tbody></table>
